Provisional name for a byte that seems to influence whether or not a character can be healed. This byte is located:
* `+0x1a3` from the true start of the character's section,
* `+0x107` from the character's name,
* `+0x6f` from the character's modified stats subsection.
## Mechanics
When Ryu transforms into a Dragon, this byte is first set to `0x03` by the following piece of code:
```
; BIN/BMAGIC/MAGIC211.EMI starting at 00016200
801e7200 lbu     $v0, 0x006f(v1) ; load the current healability byte
801e7204 nop   
801e7208 ori     $v0, 0x0003 ; ors the current healability byte with 0x03
801e720c j       0x801e7224
801e7210 sb      $v0, 0x006f(v1) ; stores the healability value
```
Then it is set to `0x01` by the following piece of code:
```
; BIN/BATTLE/BTLMOVE.EMI starting at 0000b014
801d3014 andi    $v0, $a0, 0x00fd ; ands the current healability value with 0xfd
801d3018 lhu     $v1, 0x00ae(a1)
801d301c nop    
801d3020 andi    $v1, 0x8000
801d3024 beq     $v1, $r0, 0x801d3098
801d3028 sb      $v0, 0x01a3(a1) ; stores the healability value
```
When Ryu reverts to his normal form, it is undone by the following piece of code:
```
801c08b0 andi    $v1, 0x00fe
801c08b4 sb      $v1, 0x01a3(v0)
```
