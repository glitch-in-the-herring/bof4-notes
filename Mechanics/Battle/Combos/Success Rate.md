Success rate is computed for the second and third attacks connecting. 
## First -> second attacks connecting
The formula used to determine whether or not the first and second attacks connect is:
$$
\begin{cases} 
\mathrm{success} & \mathrm{random}(0, 99) - 85 <0\\
\mathrm{failure} & \mathrm{random}(0, 99) - 85 >0\\
\end{cases}
$$
In assembly this is
```
801b25e8 li      $s2, 0x0055 ; the success rate of the two attacks connecting
...
801b2620 addiu   $s2, 0x000a ; if a harmonic ring is equipped then 10pt is added to the success rate
...
801b2790 jal     0x8016f384 ; call the rand() function
...
801b2798 li      $v1, 0x51eb851f
801b27a0 mult    $v0, $v1 ; at this point, $v0 has the random integer. It is multiplied with 0x51eb851f, the magic number used to divide by 0
...
801b27c4 mfhi    $a3, $hi
801b27c8 sra     $a0, $a3, 0x05 ; with the mult instruction above, this divides the random number by 100, rounded down
801b27cc subu    $a0, $v1
801b27d0 sll     $v1, $a0, 0x01 ; random number // 10 * 2
801b27d4 addu    $v1, $a0 ; random number // 10 * 3
801b27d8 sll     $v1, 0x03 ; random number // 100 * 24
801b27dc addu    $v1, $a0 ; random number // 100 * 25
801b27e0 sll     $v1, 0x02 ; random number // 100 * 25 * 4 = random number // 100 * 100. This is more or less rounding the number to the smallest multiple of 100.
801b27e4 subu    $v0, $v1 ; random number - random number // 100 * 100. This gives an integer between 0 and 99.
801b27e8 andi    $v1, $s2, 0x00ff
801b27ec subu    $v0, $v1 ; \mathrm{random}(0,100) - success rate
801b27f0 lw      $s2, 0x0018(sp)
801b27f4 srl     $v0, 0x1f ; shift \mathrm{random}(0,100) - success rate all the way to the right, leaving only the sign byte 
801b27f8 jr      $ra
801b27fc addiu   $sp, 0x0030
801b24e8 andi    $v0, 0x00ff
801b24ec beq     $v0, $r0, 0x801b227c ; if the sign is negative, then perform the combo.
```
In other words, the probability of the first and second attacks connecting is 85%.

## Second -> third attacks connecting
The formula used to determine whether or not the second and third attacks connect is:
$$
\begin{cases}
\mathrm{success} & \mathrm{random}(0, 99) - 75 <0\\
\mathrm{failure} & \mathrm{random}(0, 99) - 75 >0\\
\end{cases}
$$
The assembly of this formula is mostly identical except for the part where the code loads the success rate:
```
801b25d0 li     $s2, 0x004b ; the success rate of the two attacks connecting
```
And the part where the game checks the roll value:
```
801b2b4c andi    $v0, 0x00ff
801b2b50 beq     $v0, $r0, 0x801b2988  if the sign is negative, then perform the combo.
```
In other words, the probability of the second and third attacks connecting is 75%, 10pt lower than the first and second attack connecting.
## Modifiers
When the [[Harmonic Ring]] is equipped, the chance of the attacks connecting increases by 10pt.
